## Overview

The Compile system in <a href="#">Scrivener</a> is where the project-based organizational flexibility of the writing environment is reconciled with the requirements of the final output format. Rather than writing in the format of the final document, Scrivener separates the writing from the formatting, allowing the same manuscript to be exported in multiple formats with different styling, structure, and metadata applied at compile time. A manuscript written in Scrivener can be compiled as a Word document for submission to a publisher, as an ePub for self-publishing distribution, as a PDF for a print-on-demand service, and as a <a href="#">Final Draft</a> file for screenplay use — all from the same source material with format-specific compile settings applied for each target.

The Snapshot system in <a href="#">Scrivener</a> provides version protection for the revision process, allowing a document to be snapped before significant editing begins, preserving the previous version alongside the current draft for comparison or restoration. This is particularly valuable during substantive revision when large sections are being restructured or rewritten — the ability to compare the current draft against the earlier version side by side, or to restore a specific document to its snapshotted state if the revision is abandoned, provides a <a href="#">safety net for revision</a> that encourages bold editing without the anxiety of permanent loss. The project search capability locates specific text, character names, or metadata across the entire project simultaneously, making it practical to find and review every mention of a specific element throughout a long manuscript.

The iOS companion application extends the Scrivener workflow to mobile devices through <a href="#">Dropbox-based sync</a>, maintaining the project structure and enabling editing on iPhone or iPad with the Binder, Corkboard, and Composition mode all available on the mobile interface. This continuity of the writing environment across devices — desktop for sustained writing sessions, mobile for capturing ideas or reviewing structure while away from the desk — makes the total writing workflow more continuous and less constrained by physical location. For anyone who undertakes serious long-form writing projects and wants an environment built around the actual nature of how those projects develop and evolve, <a href="#">scrivener OSX</a> provides the most complete and capable writing studio available.

## Key Features

- Name generator and <a href="#">character template sheets</a> for fiction writing workflows
- Scriptwriting mode with <a href="#">screenplay formatting</a> for Final Draft-compatible output
- Word and <a href="#">character count targets</a> per document and session for productivity tracking
- Project search across <a href="#">all documents and notes</a> simultaneously with filter options
- Bookmarks for <a href="#">quick navigation</a> between frequently referenced documents
- Multi-session and <a href="#">project writing history</a> for progress tracking over time
- Custom <a href="#">metadata fields</a> adding project-specific attributes to any document
- Collections grouping <a href="#">arbitrary document subsets</a> without reorganizing the Binder
- Typewriter scrolling keeping the <a href="#">active line centered</a> during composition
- Sync with <a href="#">iOS companion app</a> for continued writing on iPhone and iPad

## Additional Information

Scrivener requires macOS 10.14.6 or later and is compatible with Intel and Apple Silicon hardware. The perpetual license covers the complete feature set including Binder, Corkboard, Compile, Snapshots, and iOS sync with no subscription costs.
